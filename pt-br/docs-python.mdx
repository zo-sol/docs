## Conceitos Principais

Estes sĂŁo os conceitos-chave que vocĂȘ precisa conhecer antes de usar o SDK IQLabs.

---

### Armazenamento de Dados (Code In)

Ă assim que vocĂȘ armazena qualquer dado (arquivos, texto, JSON) on-chain.

#### Como os dados sĂŁo armazenados?

Dependendo do tamanho dos dados, o SDK escolhe o mĂ©todo ideal:

- **Dados pequenos (&lt; 850 bytes)**: armazenamento imediato, mais rĂĄpido
- **Dados mĂ©dios (&lt; 8.5 KB)**: divididos em mĂșltiplas transaĂ§Ă”es
- **Dados grandes (&gt;= 8.5 KB)**: upload em paralelo para maior velocidade

#### FunĂ§Ă”es relacionadas principais

- [`code_in()`](#code_in): enviar dados e obter um ID de transaĂ§ĂŁo
- [`read_code_in()`](#read_code_in): ler dados a partir de um ID de transaĂ§ĂŁo

---

### PDA de Estado do UsuĂĄrio

Uma conta de perfil on-chain para um usuĂĄrio.

#### O que Ă© armazenado?

- InformaĂ§Ă”es do perfil (nome, foto de perfil, bio, etc.)
- NĂșmero de arquivos enviados
- Registros de solicitaĂ§Ă”es de amizade

<Note>
As solicitaĂ§Ă”es de amizade nĂŁo sĂŁo armazenadas como valores no PDA; elas sĂŁo enviadas como transaĂ§Ă”es.
</Note>

#### Quando Ă© criado?

Ă criado automaticamente na primeira vez que vocĂȘ chama [`code_in()`](#code_in). Nenhuma configuraĂ§ĂŁo extra Ă© necessĂĄria, mas o primeiro usuĂĄrio pode precisar assinar duas vezes.

---

### PDA de ConexĂŁo

Uma conta on-chain que gerencia relacionamentos entre dois usuĂĄrios (amigos, mensagens, etc.).

#### Quais estados ele pode ter?

- **pending** (pendente): uma solicitaĂ§ĂŁo de amizade foi enviada, mas ainda nĂŁo foi aceita
- **approved** (aprovado): a solicitaĂ§ĂŁo foi aceita e os usuĂĄrios estĂŁo conectados
- **blocked** (bloqueado): um lado bloqueou o outro

<Warning>
Uma conexĂŁo bloqueada sĂł pode ser desbloqueada por quem bloqueou.
</Warning>

#### FunĂ§Ă”es relacionadas principais

- [`request_connection()`](#request_connection): enviar uma solicitaĂ§ĂŁo de amizade (cria status pendente)
- [`manage_connection()`](#manage_connection): aprovar/rejeitar/bloquear/desbloquear uma solicitaĂ§ĂŁo
- [`read_connection()`](#read_connection): verificar o status atual do relacionamento
- [`write_connection_row()`](#write_connection_row): trocar mensagens/dados com um amigo conectado
- [`fetch_user_connections()`](#fetch_user_connections): buscar todas as conexĂ”es (solicitaĂ§Ă”es de amizade enviadas e recebidas)

### Tabelas de Banco de Dados

Armazene dados JSON em tabelas como um banco de dados.

#### Como as tabelas sĂŁo criadas?

Use [`create_table()`](#create_table) para criar uma tabela explicitamente. Se o PDA DbRoot estiver com pouco espaĂ§o, o SDK o expande automaticamente na mesma transaĂ§ĂŁo â sem etapas adicionais.

<Note>
Uma tabela Ă© identificada de forma Ășnica pela combinaĂ§ĂŁo de `db_root_id` e `table_seed` (nome da tabela).
</Note>

#### FunĂ§Ă”es relacionadas principais

- [`create_table()`](#create_table): criar uma nova tabela (expande DbRoot automaticamente se necessĂĄrio)
- [`write_row()`](#write_row): adicionar uma nova linha a uma tabela existente
- [`read_table_rows()`](#read_table_rows): ler linhas de uma tabela
- [`get_tablelist_from_root()`](#get_tablelist_from_root): listar todas as tabelas em um banco de dados
- [`fetch_inventory_transactions()`](#fetch_inventory_transactions): listar arquivos enviados

### Criptografia (Crypto)

O SDK fornece funcoes criptograficas de ponta a ponta para proteger dados on-chain.

#### Modos de criptografia

- **Criptografia DH (destinatario unico)**: X25519 ECDH efemero â HKDF-SHA256 â AES-256-GCM
- **Criptografia por senha**: PBKDF2-SHA256 (250.000 iteracoes) â AES-256-GCM
- **Criptografia multi-destinatarios (hibrido estilo PGP)**: CEK aleatoria, criptografa uma vez, envelopa a CEK por ECDH para cada destinatario

#### Derivacao de chaves

Um par de chaves X25519 e derivado a partir da assinatura da carteira via [`derive_x25519_keypair()`](#derive_x25519_keypair).

#### Funcoes relacionadas principais

- [`derive_x25519_keypair()`](#derive_x25519_keypair): derivar par de chaves de criptografia a partir da assinatura da carteira
- [`dh_encrypt()`](#dh_encrypt) / [`dh_decrypt()`](#dh_decrypt): criptografia/descriptografia para destinatario unico
- [`password_encrypt()`](#password_encrypt) / [`password_decrypt()`](#password_decrypt): criptografia/descriptografia baseada em senha
- [`multi_encrypt()`](#multi_encrypt) / [`multi_decrypt()`](#multi_decrypt): criptografia/descriptografia multi-destinatarios
